# Blueprint

Professional project blueprint for the portfolio and admin system.

## Summary
- Public portfolio reads **published** data in real time.
- Admin dashboard manages content and files (private, Access‑protected).
- Media assets stored in R2 and served via a Functions proxy.

## Stack
- Astro (server output)
- Tailwind CSS v4
- Iconify web component

## Cloudflare Services
- Pages (hosting)
- Pages Functions (API)
- D1 (content database)
- R2 (file storage)
- Zero Trust Access (admin auth)

## Bindings
- D1: `DB` → `astro-db`
- R2: `STORAGE` → `astro-r2`

## Dev Commands
```bash
npm install
npm run dev -- --host
```

Cloudflare Functions locally (proxy):
```bash
npm run dev:cf
```

## Build
```bash
npm run build
npm run preview
```

## Public Endpoints (read‑only)
- `/data/projects`
- `/data/blogs`
- `/data/tools`
- `/data/certifications`
- `/data/profile-assets`
- `/media/*` (R2 proxy)

## Private Endpoints (Access‑protected)
- `/api/projects`
- `/api/blogs`
- `/api/tools`
- `/api/certifications`
- `/api/profile-assets`
- `/api/uploads/:type`

## Data Model
- Projects: name, summary, href, status
- Blogs: title, summary, slug, content, reading_time, date, author, status
- Tools: name, icon, status
- Certifications: name, year, issued_by, credential_id, verify_link, file_key, status
- Profile: avatar_key, photo_key, resume_key, summary

## Notes
- Only items with `status = published` appear on the public site.
- `/admin` and `/api/*` are guarded by Access and `functions/_middleware.ts`.
